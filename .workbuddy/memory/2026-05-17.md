### Mastermind 游戏滑块开关修复
- **问题**：底部滑块开关只能点击一次，第二次点击无法将滑块从右侧划回左侧
- **排查过程**：
  1. 检查 CSS 选择器特异性（`.mode-switch.side-mode .mode-thumb` 分数 30 vs `.mode-thumb` 分数 10）
  2. 检查 JS 事件绑定和 `changeDisplayMode()` 函数
  3. 发现 `data-mode` 属性选择器方案存在潜在问题
  4. 尝试 CSS `!important` 方案（用户反馈无效）

- **最终方案**：JavaScript 内联样式（彻底绕过 CSS 特异性）
  ```javascript
  const thumb = modeSwitch.querySelector('.mode-thumb');
  if (mode === 'grid') {
      modeGrid.classList.add('active');
      thumb.style.left = '3px';
  } else {
      modeSide.classList.add('active');
      thumb.style.left = 'calc(50% - 3px)';
  }
  ```

- **修改文件**：
  - `/Users/douba/Projects/mima/game.js` (changeDisplayMode 函数)
  - `/Users/douba/Projects/mima/styles.css` (移除 CSS 选择器依赖)

## 经验总结
- JavaScript 直接设置 `element.style.left` 是控制 DOM 位置的最可靠方式
- CSS 选择器优先级在动态 UI 切换中可能产生不可预期的行为
- 保留 CSS `transition` 属性用于动画效果，位置由 JS 控制
